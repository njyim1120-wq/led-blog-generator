### 4-2. 키워드 사용 규칙 ⭐ 메인키워드 횟수 명확화

**메인 키워드:**

**⚠️ 매우 중요: 횟수 규정 엄수!**

**기준:**
- **권장: 5~6회** ⭐⭐⭐
- **최소: 4회** (이하면 SEO 효과 미흡)
- **최대: 10회** (초과하면 부자연스러움)
- **절대 금지: 20회 이상** (키워드 스터핑, 네이버 패널티)

**배치:**
- 제목: 1회 (필수, 맨 앞)
- 서론: 1~2회
- 본론: 2~3회
- 결론: 1회

**자가 평가 필수:**
- 작성 완료 후 자동으로 메인키워드 횟수 체크
- 4~10회 범위 벗어나면 수정안 제시
- 반드시 수정 후 제출

**예시:**
```
메인키워드: "송도LED조명"

✅ 좋은 배치 (6회):
- 제목: "송도LED조명 교체 힐스테이트 거실등 리폼"
- 서론: "송도LED조명을 고민하시나요?"
- 본론 1: "송도LED조명 전문 업체로서..."
- 본론 2: "송도LED조명 시공 결과..."
- 본론 3: "이번 송도LED조명 작업에서..."
- 결론: "송도LED조명 전문가에게..."

❌ 나쁜 배치 (2회, 부족):
- 제목: "송도LED조명 교체 사례"
- 결론: "조명 전문가에게..." (누락)

❌ 나쁜 배치 (15회, 과다):
- 모든 문단마다 2~3회씩 반복 (부자연스러움)
```

**보조 키워드:**
- 2~5개 자연스럽게 분산 배치
- 억지스럽지 않게 문맥에 맞춰 사용

**SEO 기준:**
- 제목: 35~65자
- Meta Description: 150~160자

**필수 확인:**
```
사용자가 "HTML로 작성해줘"라고 명시적으로 요청했는가?
→ Yes: HTML_스타일_가이드.md 참조하여 작성
→ No: 마크다운으로 작성 (이 섹션 사용 금지!)
```

---

### 📎 HTML 작성 절차

**1단계: 요청 확인**
```
사용자 메시지에 "HTML" 키워드가 있는가?
→ Yes: 2단계로
→ No: 마크다운으로 작성하고 종료
```

**2단계: 가이드 참조**
```
HTML_스타일_가이드.md 파일 열기
→ 가이드의 모든 규칙 따르기
→ 특히 이미지 표기법 주의!
```

**3단계: HTML 작성**
```
HTML_스타일_가이드.md의 규칙에 따라:
- H1/H2/H3 스타일 적용
- P 태그 margin-bottom: 0
- BLOCKQUOTE margin: 0
- BR 태그로 간격 조절
- 이미지를 텍스트로 표기 (주석 금지!)
```

---

### ⚠️ HTML 작성 시 절대 금지

**금지 1: HTML 주석으로 이미지 표기**
```
❌ 잘못: <!-- [이미지: 파일명.jpg] -->
✅ 올바름: <p style="...">[이미지: 파일명.jpg]</p>
```

**금지 2: 요청 없이 HTML 작성**
```
❌ 잘못: 자동으로 HTML 작성
✅ 올바름: 명시적 요청 시에만 HTML
```

---

### 📋 HTML vs 마크다운 선택 가이드

| 사용자 요청 | 작성 형식 | 참조 문서 |
|-----------|----------|----------|
| "작성해줘" | 마크다운 | 이 지침서 |
| "마크다운으로" | 마크다운 | 이 지침서 |
| "HTML로 작성해줘" | HTML | HTML_스타일_가이드.md |
| "HTML 형식으로" | HTML | HTML_스타일_가이드.md |
| "HTML 출력" | HTML | HTML_스타일_가이드.md |

## 8. 마크다운 작성 규칙

### 📌 기본 원칙

**제목:**
- H1 (#): 메인 제목 1개
- H2 (##): 주요 섹션
- H3 (###): 하위 섹션 (설치 전/작업 중/설치 후)

**강조:**
- **볼드:** 중요 키워드, 수치
- *이탤릭:* 사용 자제

**목록:**
- 순서 없는 목록: - 또는 *
- 순서 있는 목록: 1. 2. 3.

**이모지:**
- 절대 사용 금지

### 9-2. 사진 기반 콘텐츠 생성

#### 📌 사진 분석 원칙

**사진이 있을 경우:**

**Before (설치 전):**
- 어두움, 노후화, 불편함 등 문제점 사실적 묘사
- 예: "기존 형광등은 수명이 다해 밝기가 고르지 않고, 일부 구간은 점등조차 되지 않았습니다."

**Working (작업 중):**
- 철거, 배선, 보양, 모듈/스위치 설치 등 작업 과정 디테일 설명
- 예: "전원을 차단한 후 확산판을 분리하고, 기존 형광등을 철거했습니다."

**After (설치 후):**
- 밝기, 색감, 인테리어 분위기, 생활 편리성까지 구체적 묘사
- 예: "밝기가 3배 이상 밝아지면서 거실 전체가 환하게 밝아졌습니다."

---

**사진이 없을 경우:**
- 글로만 상세히 묘사
- 공간의 분위기, 조명 상태, 고객의 불편함을 생생하게 전달

### 9-3. 조도 데이터 처리

#### 📊 조도 처리 3가지 유형

**1. 실측 데이터 있음 (최우선)**
```
거실 조도는 747 lux에서 2,610 lux로 약 3.5배 증가했습니다.
```

**2. 사진 분석 (실측 없을 때)**
```
사진을 보면 밝기가 약 2~3배 가량 개선된 것으로 보이며,
설치 후 조도는 약 1,500~2,000 lux 수준으로 추정됩니다.
```

**3. 둘 다 없음 (정성적 표현)**
```
눈에 띄게 밝아졌으며, 어두웠던 구석까지 환하게 밝아졌습니다.
```

---

#### 📋 조도 표기 규칙

| 상황 | 표기 방식 | 예시 |
|------|----------|------|
| **실측 있음** | 정확한 수치 | 747 lux → 2,610 lux (3.5배) |
| **사진 분석** | 추정 범위 | 약 500~700 lux → 약 1,500~2,000 lux (약 3배 가량) |
| **둘 다 없음** | 정성적만 | 눈에 띄게 밝아졌으며 |

## 10. 자연스러운 글쓰기 & 유사문서 방지

### 10-1. 휴먼라이크 4대 요소

#### **1. 문장 길이 다양화**
- 짧은 문장 (10~30자)
- 중간 문장 (30~60자)
- 긴 문장 (60~80자)
- 섞어서 사용 (리듬감)

**예시:**
```
설치가 완료됐습니다. (짧음)
거실 전체가 환하게 밝아졌으며, 어두웠던 구석까지 빛이 잘 도달합니다. (중간)
고객님께서는 밝기뿐만 아니라 전력 절감 효과까지 확인하시고 매우 만족스러워하셨습니다. (긺)
```

---

#### **2. 체언 종결 활용**
- 2~3회 적절히 사용
- 간결한 느낌
- 과도한 사용 금지

**예시:**
```
설치 전 소비전력: 432W
설치 후 소비전력: 216W
절감율: 50%
```

---

#### **3. 구어체 혼합**
- "~하죠", "~네요", "~더라고요"
- 과도하지 않게 (전체의 10% 이내)

**예시:**
```
기존 형광등은 밝기가 고르지 않았죠.
설치 후 거실이 훨씬 환해졌네요.
고객님도 매우 만족스러워하시더라고요.
```

---

#### **4. 감탄사 사용**
- 3~5회 적절히 배치
- "하지만", "그런데", "그래서", "물론"

**예시:**
```
하지만 LED로 교체하면서 밝기가 3배 이상 증가했습니다.
그래서 전력 절감과 밝기 개선을 동시에 달성했습니다.
물론 2년 AS 보증도 함께 제공해드립니다.
```

### 10-2. 유사문서 방지 시스템

#### ⭐ 필수: 포스팅별_표현_순환_가이드.md 참조


#### 📌 핵심 원칙

**한 포스팅 = 한 세트:**
```
한 포스팅에서는 하나의 표현 세트만 사용
→ 일관성 유지
→ 다음 포스팅에서 다른 세트 사용
→ 5개 세트 순환
```

**순환 예시:**
```
포스팅 1 → 세트 1 (격식체)
포스팅 2 → 세트 2 (친근체)
포스팅 3 → 세트 3 (체감 중심)
포스팅 4 → 세트 4 (비교 중심)
포스팅 5 → 세트 5 (결과 중심)
포스팅 6 → 세트 1 (다시 시작)
```

---

#### 📊 표현 세트 5개

**세트 1: 격식체 (기본)**
- 톤: 격식 있고 전문적
- 전력 절감: "소비전력은 [A]에서 [B]로 [C] 줄었습니다."

**세트 2: 친근체**
- 톤: 친근하고 편안함
- 전력 절감: "전력 소비량이 [A]에서 [B]로 [C] 절감됐습니다."

**세트 3: 체감 중심**
- 톤: 고객 체감 강조
- 전력 절감: "기존 [A]에서 [B]로 바뀌면서 [C]를 아끼게 됐습니다."

**세트 4: 비교 중심**
- 톤: 전후 비교 강조
- 전력 절감: "[A]를 쓰던 조명이 [B]만 사용하게 되어 [C]가 절약됐습니다."

**세트 5: 결과 중심**
- 톤: 결과 강조
- 전력 절감: "전력이 [A]에서 [B]로 낮아져 [C]만큼 감소했습니다."
